## 용어 설명

| 쉬운 말 | 전문 용어 | 뜻 |
|---------|-----------|-----|
| 서버 코드 | Backend / FastAPI | API를 만드는 Python 프로그램 |
| 관리 화면 코드 | Frontend / Next.js | 브라우저 화면 |
| 설정값 | 환경변수 | `.env` 파일에 적는 키=값 |
| 테이블 버전 관리 | Alembic | DB 구조를 안전하게 바꾸는 도구 |
| 관리 열쇠 | `ADMIN_API_KEY` / `X-Admin-API-Key` | 위험한 API를 잠그는 비밀번호 |
| 브로커 | Broker | 증권사·거래소 연결 (키움 등) |
| 웹소켓 | WebSocket | 실시간으로 계속 주고 받는 연결 |

## 2. 폴더가 어디에 있는지

```text
stock-platform/
├── src/stock_platform/   ← 서버(백엔드) 코드
├── frontend/             ← 관리 화면 코드
├── database/alembic/     ← DB 테이블 만들기 스크립트
├── scripts/              ← 스케줄러·점검용 스크립트
├── docs/manual/          ← 지금 보고 있는 매뉴얼
├── tests/                ← 자동 시험
├── alembic.ini           ← Alembic 설정
├── requirements.txt      ← Python 패키지 목록
└── .env.example          ← 설정 예시(비밀 아님)
```

| 쉬운 이름 | 경로 | 하는 일 |
|-----------|------|---------|
| API | `src/stock_platform/api/` | 웹 주소(엔드포인트) 모음 |
| 증권 연동 | `src/stock_platform/broker/` | 키움 등 |
| 실시간 | `src/stock_platform/realtime/` | 실시간 전략·체결 |
| 운영 기록 | `src/stock_platform/operation/` | 감사·헬스 |
| 화면 | `frontend/src/` | Admin UI |
| **실행용** DB 변경 | `database/alembic/versions/` | 마이그레이션 |
| 참고만 | `docs/migration-overlays/` | **실행 금지** |

**실행 명령 요약**

| 목적 | 명령(개념) |
|------|------------|
| 서버 | `uvicorn stock_platform.api.main:app` |
| 관리 화면 | `frontend` 에서 `npm run dev` |
| 스케줄러 | `scripts/run_scheduler.py` |

**[스크린샷]** 탐색기에서 프로젝트 최상위 폴더

## 3. 설정 파일·비밀번호 관리

| 무엇 | 어디 |
|------|------|
| 서버 비밀 설정 | `E:\StockTrading\secrets\stock-platform.env` (기본 로드 경로) |
| 작성 참고 | 프로젝트 `.env.example` |
| 관리 화면 설정 | `frontend/.env.local` |
| 설명 문서 | [../deployment/CONFIGURATION.md](../deployment/CONFIGURATION.md) |

### 자주 쓰는 설정 (쉬운 설명)

| 설정 이름 | 쉬운 설명 |
|-----------|-----------|
| `DB_*` | 데이터베이스 주소·이름·비밀번호 |
| `JWT_SECRET` | 로그인 토큰 서명 (필수) |
| `AUTH_BOOTSTRAP_ADMIN_*` | 최초 admin 계정 (사용자 0명일 때만) |
| `KIWOOM_USE_MOCK` | true면 키움 연습 모드 |
| `KIWOOM_LIVE_ORDER_ENABLED` | true면 실전 주문 허용(기본 false) |
| `KIWOOM_APP_KEY` / `SECRET` | 키움 앱 인증 |
| `KIWOOM_ACCOUNT_NUMBER` | 계좌번호 |
| `OLLAMA_BASE_URL` / `MODEL` | 로컬 AI 주소·모델 이름 |
| `UPBIT_*` | 업비트 접속 |
| `DART_API_KEY` | 공시(DART) |
| `NAVER_*` | 뉴스 수집 |
| `ADMIN_API_KEY` | 관리 API 열쇠 (`X-Admin-API-Key`) |
| `SCHEDULER_ENABLED` | 스케줄러 사용 여부 |
| `TELEGRAM_*` / `SLACK_*` / `DISCORD_*` | 알림 |

런타임 튜닝(비밀 아님)은 Admin **시스템설정 / 환경설정** UI (`/api/v1/settings`)에서도 관리합니다.

**[스크린샷]** 설정 파일 위치(비밀번호는 가리고)

## 5. 스케줄러(자동 작업)

### (1) 서버와 함께 도는 타이머

서버를 켜면 같이 시작되고, 서버를 끄면 같이 멈춥니다.  
예: 일일 손실 감시, 전략 상태 다시 읽기 등

### (2) 별도 스케줄러 프로그램

| 항목 | 내용 |
|------|------|
| 파일 | `scripts/run_scheduler.py` |
| 조건 | `SCHEDULER_ENABLED=true` |
| 한 번만 실행 | `scripts/run_scheduler_job.py 작업이름` |
| Windows 등록 | `install_scheduler_task.ps1` / `uninstall_...` |
| API로 즉시 실행 | `POST /api/v1/scheduler-admin/run-now/{작업이름}` + 관리 키 |

작업 이름 예: `candidate_screening`, `ai_orchestration`, `position_planning` …

### Worker / Redis?

별도의 Redis 대기열 워커는 **없습니다.**  
시세 저장 등은 서버 안에서 처리합니다. (`/health` 의 queue 항목 참고)

더 자세한 켜기/끄기 → [운영매뉴얼.md](운영매뉴얼.md)

## 9. 권한·관리 키

| 계층 | 지금 상태 |
|------|-----------|
| 관리 화면 로그인 | **JWT** — `/login` (회원·역할·권한 RBAC) |
| 최초 관리자 | `AUTH_BOOTSTRAP_ADMIN_USERNAME` / `PASSWORD` (사용자 0명일 때만) |
| 관리 API | 헤더 `X-Admin-API-Key` = `ADMIN_API_KEY` **또는** admin JWT |
| 키가 비어 있을 때 | Admin Key 검사는 로컬에서 통과 가능. **운영에서는 반드시 키·JWT_SECRET 설정** |

역할 예: `admin` / `operator` / `viewer` — Admin **권한관리** (`/admin/roles`)

### 관리 키가 필요한 작업 (예)

- 감사 로그 조회
- 스케줄 즉시 실행
- 주문 dispatch
- Kill Switch 켜기/끄기
- 실전 전환 요청·승인·해제
- 전략 정책 강제/거절

목록 전체는 [API사용매뉴얼.md](API사용매뉴얼.md) 참고
