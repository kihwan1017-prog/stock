## 프로젝트 구조

```text
stock-platform/
├── src/stock_platform/     # Backend 패키지 (FastAPI)
├── frontend/               # Admin Next.js (STEP41)
├── database/alembic/       # Alembic migrations
├── scripts/                # 스케줄러·검증·작업 스크립트
├── docs/                   # 문서 (manual/ 포함)
├── tests/                  # pytest
├── alembic.ini
├── requirements.txt
└── .env.example
```

진입점:

| 구분 | 명령 / 경로 |
|------|-------------|
| API | `uvicorn stock_platform.api.main:app` (`src`를 PYTHONPATH) |
| Admin | `frontend/` → `npm run dev` |
| 스케줄러 | `scripts/run_scheduler.py` |

## Scheduler / Worker

### 1) API lifespan 내부 작업

uvicorn 기동 시 `application_lifecycle`이 DB 점검·(설정 시) recovery·내부 스케줄·시세 영속화 등을 수행합니다.  
API 프로세스와 함께 시작하고 종료됩니다.

### 2) AutomaticScheduler

- 스크립트: `scripts/run_scheduler.py`
- 조건: `SCHEDULER_ENABLED=true`
- 수동 Job: `scripts/run_scheduler_job.py <job_name>`
- Windows 등록: `scripts/install_scheduler_task.ps1` / `uninstall_scheduler_task.ps1`
- Admin API: `POST /api/v1/scheduler-admin/run-now/{job_name}` (`X-Admin-API-Key`)

등록 Job 예: `candidate_screening`, `ai_orchestration`, `position_planning` 및 레지스트리 Job(`daily_operations_report`, `upbit_krw_daily_sync`, `indicator_daily_batch` 등)

### 3) Worker

별도 Redis 큐 워커 프로세스는 **없습니다**.  
시세 영속화 등은 API lifespan 쪽 컴포넌트로 동작합니다 (`/health`의 `queue` 상태 참고).

## 권한

| 계층 | 구현 |
|------|------|
| Admin 웹 JWT 역할 | **없음** (`AUTH_MODE=disabled`) |
| 관리 API | `X-Admin-API-Key` ↔ `ADMIN_API_KEY` |
| 키 미설정 | `require_admin`이 `"DEV_OPEN"`으로 **통과** (로컬 전용) |
| 운영 | `ADMIN_API_KEY` 필수 설정 |

Admin 보호 엔드포인트 예:

- `GET /api/v1/audit/events`
- `POST /api/v1/scheduler-admin/run-now/{job_name}`
- `POST /api/v1/orders/{order_id}/dispatch`
- `POST /api/v1/risk/kill-switch/activate|deactivate`
- `POST /api/v1/broker/live-transition/request|.../approve|.../disable`
- `GET /api/v1/broker/live-transition/history`
- `POST /api/v1/strategy-policy/{id}/force|reject`
