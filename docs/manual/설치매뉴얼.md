## 1. 필요한 프로그램 설치

| 순서 | 프로그램 | 왜 필요한가 |
|------|----------|-------------|
| 1 | **Git** | 소스 코드를 받아 오기 위해 |
| 2 | **Python 3.12 이상** | 서버(백엔드) 실행 |
| 3 | **Node.js 20 이상** | 관리 화면 실행 |
| 4 | **PostgreSQL 17** | 데이터 저장 (Windows 서비스로 실행) |
| 5 | **Ollama** | AI 기능·건강 점검에 사용 |

**설치하지 않아도 되는 것:** Docker, Redis

각 프로그램 설치 창이 뜨면 기본값으로 진행해도 됩니다.  
PostgreSQL은 **설치할 때 정한 비밀번호를 꼭 메모**하세요.

**[스크린샷]** PostgreSQL 설치 완료 / 서비스가 실행 중인지 보이는 화면

## 2. 프로젝트 받아오기

PowerShell(또는 터미널)을 엽니다.

```powershell
git clone https://github.com/kihwan1017-prog/stock.git
cd stock
```

이미 `D:\Projects\stock-platform` 에 있다면 그 폴더로 이동하면 됩니다.

```powershell
cd D:\Projects\stock-platform
```

**[스크린샷]** 프로젝트 폴더가 열린 탐색기 또는 터미널

---

## 3. Python 환경 만들기

프로젝트 폴더에서:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

- `.venv` = 이 프로젝트 전용 Python 상자  
- `requirements.txt` = 필요한 라이브러리 목록

실행 정책 때문에 Activate가 막히면, 관리자에게 문의하거나  
`Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` 후 다시 시도하세요.

**[스크린샷]** 터미널 앞에 `(.venv)` 가 보이는 상태

## 4. 비밀 설정 파일 만들기

1. 프로젝트 안의 `.env.example` 을 참고합니다.  
2. 실제 비밀 파일은 보통 여기에 둡니다.  
   `E:\StockTrading\secrets\stock-platform.env`  
   (프로그램이 기본적으로 이 경로를 읽습니다.)
3. 최소한 **데이터베이스 접속 정보**를 채웁니다.  
   (`DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`)

처음에는 아래처럼 **안전하게** 두는 것을 권장합니다.

```dotenv
KIWOOM_USE_MOCK=true
KIWOOM_LIVE_ORDER_ENABLED=false
SCHEDULER_ENABLED=true
ADMIN_API_KEY=
```

| 설정 | 쉬운 설명 |
|------|-----------|
| `KIWOOM_USE_MOCK=true` | 키움을 연습(모의) 모드로 |
| `KIWOOM_LIVE_ORDER_ENABLED=false` | 실전 주문 **금지** |
| `SCHEDULER_ENABLED=true` | 정해진 시간에 돌아가는 작업 허용 |
| `ADMIN_API_KEY=` | 비우면 이 PC에서 관리 API가 열림(나중에 운영 시 반드시 설정) |

자세한 목록: [../deployment/CONFIGURATION.md](../deployment/CONFIGURATION.md)

**[스크린샷]** 비밀 설정 파일 위치(비밀번호는 가리고 촬영)

## 5. 데이터베이스 만들기

1. PostgreSQL에서 **데이터베이스 이름**과 **사용자**를 만듭니다.  
2. 4번의 `DB_NAME` / `DB_USER` / `DB_PASSWORD` 와 **똑같이** 맞춥니다.  
3. 연결 시험:

```powershell
.\.venv\Scripts\python.exe scripts\test_db_connection.py
```

성공 메시지가 나오면 다음 단계로 갑니다.

## 6. 테이블 만들기 (마이그레이션)

**마이그레이션** = DB 안에 필요한 표(테이블)를 자동으로 만드는 과정입니다.  
도구 이름: **Alembic**

```powershell
$env:PYTHONPATH = "D:\Projects\stock-platform\src"
.\.venv\Scripts\python.exe -m alembic upgrade head
.\.venv\Scripts\python.exe -m alembic heads
```

추가 확인(선택):

```powershell
.\scripts\verify_alembic.ps1
```

> `docs/migration-overlays` 폴더의 파일은 **참고용**입니다.  
> 여기서 실행하지 마세요.

**[스크린샷]** `alembic upgrade head` 성공 출력

## 7. 서버(백엔드) 켜 보기

```powershell
$env:PYTHONPATH = "D:\Projects\stock-platform\src"
.\.venv\Scripts\uvicorn.exe stock_platform.api.main:app --host 127.0.0.1 --port 8000
```

브라우저에서 확인:

| 주소 | 의미 |
|------|------|
| http://127.0.0.1:8000/health | 건강 점검 |
| http://127.0.0.1:8000/docs | API 설명서 |

**[스크린샷]** `/health` 결과  
**[스크린샷]** `/docs` (Swagger) 화면

이 창은 **끄지 말고** 두고, 새 터미널을 열어 다음 단계를 진행합니다.

## 8. 관리 화면(프론트) 설치·켜 보기

새 PowerShell:

```powershell
cd D:\Projects\stock-platform\frontend
npm install
copy .env.example .env.local
npm run dev
```

`.env.local` 기본 예:

```dotenv
NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000
NEXT_PUBLIC_API_PREFIX=/api/v1
NEXT_PUBLIC_AUTH_MODE=disabled
```

- `AUTH_MODE=disabled` → 아이디 없이 「개발 모드로 입장」 사용

브라우저: http://localhost:3000

**[스크린샷]** 관리 화면 로그인(입장) 페이지

더 자세한 프론트 안내: [../../frontend/README.md](../../frontend/README.md)

## 9. 설치가 잘 됐는지 확인

| 확인 | 방법 |
|------|------|
| 서버 | `/health` 가 열림 |
| API 설명서 | `/docs` 가 열림 |
| 관리 화면 | `http://localhost:3000` 입장 가능 |
| (선택) 자동 점검 | `.\scripts\verify_release.ps1` |

모두 되면 설치 완료입니다.

## 10. 계정은 어떻게 되나요?

| 구분 | 상태 |
|------|------|
