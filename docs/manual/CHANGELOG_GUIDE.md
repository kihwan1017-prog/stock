## 용어 설명

| 쉬운 말 | 전문 용어 | 뜻 |
|---------|-----------|-----|
| 변경 일기 | CHANGELOG | 버전별 바뀐 점 목록 |
| 아직 안 낸 변경 | Unreleased | 다음 버전에 들어갈 예정인 항목 |
| 버전 딱지 | Git tag | 예: `v1.0.0` |

**[스크린샷]** CHANGELOG.md 상단 (Unreleased / 최신 버전)

---

## 2. 어떻게 읽나요?

1. **위에서부터** 읽습니다. (가장 최근이 위)  
2. `Unreleased` = 아직 버전 번호를 안 붙인 변경  
3. 그 아래 `1.0.0 — 날짜` 같은 제목이 **공식 출시 내용**  
4. 소제목 의미:
   - **Added** = 새로 생김  
   - **Changed** = 기존이 바뀜  
   - **Fixed** = 고장 고침  
   - **Security** = 보안·실전 주문 관련 (중요)  
   - **Docs** = 문서만 변경  

운영에 특히 중요한 것: **Security**, 실전 주문, DB 구조 변경

---

## 3. 관리자가 새로 적을 때

새 버전을 내기 전에 `Unreleased` 내용을 버전 칸으로 옮깁니다.

```markdown
## Unreleased

### Docs
- ...

## 1.0.1 — 2026-07-20

### Added
- ...
```

적을 때:

- **왜** 바뀌었는지 한 줄로 (사용자 입장)
- 비밀번호·개인정보 금지
- 매뉴얼을 고쳤으면 `docs/manual/` 링크를 Docs에 남기기

---

## 4. 매뉴얼과 함께 맞추기

기능이 추가되면:

1. 프로그램·시험 반영  
2. `/docs`(OpenAPI) 확인  
3. 해당 매뉴얼 절 수정 (**추측 금지**)  
4. `CHANGELOG.md` 에 한 줄 추가
