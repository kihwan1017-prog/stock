## 초기 점검

1. `GET http://127.0.0.1:8000/health`
2. PostgreSQL Windows 서비스 상태
3. Ollama 프로세스 (`OLLAMA_BASE_URL`)
4. `KIWOOM_LIVE_ORDER_ENABLED`가 의도치 않게 `true`인지
5. Kill Switch: `GET /api/v1/risk/kill-switch`

## 자주 발생하는 문제

### DB 연결 실패

| 증상 | 조치 |
|------|------|
| `/health` → `components.database` DOWN | 서비스 시작, `DB_*` 확인 |
| Alembic/앱 기동 오류 | `scripts\test_db_connection.py` |
| multiple heads | `alembic heads` 후 chain 정리 |

### 키움 로그인·연동 실패

| 증상 | 조치 |
|------|------|
| 토큰 실패 | `POST /api/v1/kiwoom/token/test`, `KIWOOM_APP_KEY/SECRET` |
| 모의만 동작 | `KIWOOM_USE_MOCK` 확인 |
| 실주문 거부 | `KIWOOM_LIVE_ORDER_ENABLED`, live-transition 승인 이력 |
| 계좌 불일치 | `KIWOOM_ACCOUNT_NUMBER` vs 스냅샷 계좌번호 형식 |

### 업비트 연결 실패

| 증상 | 조치 |
|------|------|
| 시세/동기화 실패 | `UPBIT_BASE_URL`, 네트워크, `GET /api/v1/upbit/markets` |
| 실시간 중단 | `/api/v1/realtime-quotes/status`, upbit start 재시도 |

### LLM / Ollama 오류

| 증상 | 조치 |
|------|------|
| `/health` ollama DOWN | Ollama 실행, `OLLAMA_BASE_URL` |
| AI API 5xx | 모델명(`OLLAMA_MODEL`)·Ollama 로그 |
| Admin AI 화면 없음 | UI Coming Soon — API로 확인 |

### Scheduler 중지

| 증상 | 조치 |
|------|------|
| Job 미실행 | `SCHEDULER_ENABLED`, `run_scheduler.py` 프로세스 |
| Windows 작업 없음 | `install_scheduler_task.ps1` |
| 단건 재실행 | `POST /api/v1/scheduler-admin/run-now/{job}` 또는 `run_scheduler_job.py` |

### 관리 API 401

- `ADMIN_API_KEY`와 `X-Admin-API-Key` 일치 여부
- 키가 비어 있으면 로컬은 통과, 운영은 반드시 설정

### 주문이 안 나감

1. Kill Switch active  
2. Outbox PENDING/FAILED (`/api/v1/order-outbox`)  
3. LIVE면 `live-transition/active`  
4. `KIWOOM_LIVE_ORDER_ENABLED=false`이면 실주문 차단이 정상

### 알림 미수신

- Telegram/Slack/Discord enabled + credential  
- 중복 억제(TTL)로 SKIPPED 여부
