## 1. 프로그램 켜기

### 1-1. 서버(필수)

PowerShell:

```powershell
cd D:\Projects\stock-platform
.\.venv\Scripts\Activate.ps1
$env:PYTHONPATH = "D:\Projects\stock-platform\src"
.\.venv\Scripts\uvicorn.exe stock_platform.api.main:app --host 127.0.0.1 --port 8000
```

켜지면 이 창은 **그대로 둡니다.** (끄면 서버가 멈춥니다.)

서버가 켜질 때 DB 연결 확인, (설정에 따라) 복구·내부 타이머·시세 저장 등을 시도합니다.

Ollama 앱이 실행 중인지 확인합니다.  
안 켜져 있으면 AI 관련 기능과 건강 점검의 AI 항목이 실패할 수 있습니다.

## 4. 정해진 시간 작업(스케줄러)

### 두 종류가 있습니다

| 종류 | 쉬운 설명 | 끄는 방법 |
|------|-----------|-----------|
| 서버 안의 타이머 | 서버를 켤 때 같이 시작 | 서버를 끄면 함께 멈춤 |
| 별도 스케줄러 프로그램 | `run_scheduler.py` | 그 창에서 `Ctrl+C` |

### 별도 스케줄러 켜기

설정 `SCHEDULER_ENABLED=true` 일 때만 의미가 있습니다.

```powershell
$env:PYTHONPATH = "D:\Projects\stock-platform\src"
.\.venv\Scripts\python.exe scripts\run_scheduler.py
```

**[스크린샷]** 스케줄러가 돌고 있는 터미널

### 작업 한 번만 수동 실행

```powershell
.\.venv\Scripts\python.exe scripts\run_scheduler_job.py candidate_screening
```

또는 API(관리 키 필요, 키가 비어 있으면 이 PC에서는 통과):

```http
POST /api/v1/scheduler-admin/run-now/candidate_screening
X-Admin-API-Key: (설정한 키)
```

### Windows에 “자동으로 켜지게” 등록

- 등록: `scripts\install_scheduler_task.ps1`
- 해제: `scripts\uninstall_scheduler_task.ps1`

### 스케줄러 끄기

- `run_scheduler.py` 창에서 `Ctrl+C`
- 또는 Windows 작업 스케줄러에서 중지 / uninstall 스크립트

등록되어 있는 작업 이름 예:  
`candidate_screening`(후보 선별), `ai_orchestration`(AI 흐름), `position_planning`(포지션 계획) 등

## 5. 자동매매 · 모의 · 실전

| 하고 싶은 것 | 지금 상태 |
|--------------|-----------|
| 관리 화면에서 시작/중지 | `/admin/trading` — realtime strategy/execution Start·Stop |
| 주문 등록·취소 · Paper | `/admin/orders` — Risk + Kill Switch 통과 필수 |
| 전략 배포·활성화 | `/admin/strategies` (PAPER) |
| 모의(연습) | `KIWOOM_USE_MOCK=true`, Paper API |
| 실전 주문 | `KIWOOM_LIVE_ORDER_ENABLED` (기본 **false**) + live-transition 승인 |
| 비상 정지 | `/admin/risk` 또는 `/api/v1/risk/kill-switch` |

**운영 기본 원칙:** 실전 주문을 함부로 켜지 마세요.

실전 전환 전에 볼 체크리스트:  
[../trading/LIVE_TRADING_CHECKLIST.md](../trading/LIVE_TRADING_CHECKLIST.md)

API로 다루는 방법: [API사용매뉴얼.md](API사용매뉴얼.md)
