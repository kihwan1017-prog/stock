## 백업 대상

| 대상 | 필수 |
|------|------|
| PostgreSQL DB (`DB_NAME`) | ✅ |
| 시크릿 env (`E:\StockTrading\secrets\stock-platform.env`) | ✅ (별도 안전한 위치) |
| Git 저장소 | 권장 (`git push`) |
| Admin `.env.local` | 권장 |

## 백업 절차

PostgreSQL `bin`이 PATH에 있다고 가정합니다.

```powershell
# 예: 일자별 덤프
$stamp = Get-Date -Format "yyyyMMdd_HHmmss"
$out = "E:\StockTrading\backups\stock_$stamp.dump"

# 커스텀 포맷 (복구 유연)
pg_dump -h $env:DB_HOST -p $env:DB_PORT -U $env:DB_USER -d $env:DB_NAME -Fc -f $out
```

또는 SQL:

```powershell
pg_dump -h ... -U ... -d ... -f "E:\StockTrading\backups\stock_$stamp.sql"
```

권장 주기: 일 1회 + 마이그레이션·실전 전환 직전

---

## 복구 절차

1. Backend / 스케줄러 **중지**
2. (필요 시) 빈 DB 생성 또는 기존 DB 교체
3. 복원:

```powershell
# -Fc 덤프
pg_restore -h $env:DB_HOST -p $env:DB_PORT -U $env:DB_USER -d $env:DB_NAME --clean --if-exists $out

# SQL 덤프
psql -h ... -U ... -d ... -f stock_YYYYMMDD.sql
```

4. Alembic 버전 확인:

```powershell
$env:PYTHONPATH = "D:\Projects\stock-platform\src"
.\.venv\Scripts\python.exe -m alembic current
.\.venv\Scripts\python.exe -m alembic heads
```

코드가 DB보다 앞서면 `alembic upgrade head`  
DB가 코드보다 앞서면 **코드/마이그레이션을 맞춤** (다운그레이드는 신중)

5. Backend 기동 후 `GET /health` 확인
