# 백업·복구 매뉴얼

**백업** = 나중에 되돌릴 수 있게 **복사본을 만드는 것**  
**복구** = 고장·실수 후 복사본으로 **되돌리는 것**

이 프로젝트에는 “한 방에 백업” 전용 버튼/스크립트가 **없습니다.**  
Windows에 설치된 PostgreSQL 도구(`pg_dump`, `pg_restore`)를 사용합니다.

## 용어 설명

| 쉬운 말 | 전문 용어 | 뜻 |
|---------|-----------|-----|
| 덤프 파일 | dump | DB를 파일로 뽑아 둔 복사본 |
| 덤프 만들기 | `pg_dump` | PostgreSQL 백업 명령 |
| 덤프 넣기 | `pg_restore` / `psql` | 복사본을 DB에 다시 넣는 명령 |
| 비밀 설정 | secrets `.env` | 비밀번호·API 키가 들어 있는 파일 |

## 2. 백업하기

PostgreSQL 설치 폴더의 `bin` 이 PATH에 있어야 `pg_dump` 가 실행됩니다.

### 방법 A — 복구에 유리한 형식 (권장)

```powershell
$stamp = Get-Date -Format "yyyyMMdd_HHmmss"
$out = "E:\StockTrading\backups\stock_$stamp.dump"

pg_dump -h $env:DB_HOST -p $env:DB_PORT -U $env:DB_USER -d $env:DB_NAME -Fc -f $out
```

### 방법 B — SQL 글자 파일

```powershell
pg_dump -h $env:DB_HOST -p $env:DB_PORT -U $env:DB_USER -d $env:DB_NAME -f "E:\StockTrading\backups\stock_$stamp.sql"
```

호스트·포트·사용자·DB 이름은 설정 파일의 `DB_*` 와 같게 넣습니다.

**[스크린샷]** 백업 파일이 생긴 폴더  
**[스크린샷]** `pg_dump` 성공 메시지(또는 파일 크기 확인)

## 3. 복구하기

**순서대로** 하세요.

1. **서버·스케줄러를 끈다** ([운영매뉴얼.md](운영매뉴얼.md))  
2. 필요하면 빈 DB를 만들거나, 기존 DB를 교체할 준비를 한다  
3. 복사본을 넣는다  

### 방법 A 복구 (`-Fc` 덤프)

```powershell
pg_restore -h $env:DB_HOST -p $env:DB_PORT -U $env:DB_USER -d $env:DB_NAME --clean --if-exists $out
```

### 방법 B 복구 (SQL 파일)

```powershell
psql -h $env:DB_HOST -p $env:DB_PORT -U $env:DB_USER -d $env:DB_NAME -f stock_YYYYMMDD.sql
```

4. DB 구조 버전 확인:

```powershell
$env:PYTHONPATH = "D:\Projects\stock-platform\src"
.\.venv\Scripts\python.exe -m alembic current
.\.venv\Scripts\python.exe -m alembic heads
```

| 상황 | 할 일 |
|------|-------|
| 코드가 DB보다 최신 | `alembic upgrade head` |
| DB가 코드보다 최신 | 코드를 맞추거나, 신중히 전문가와 상의 (함부로 내리지 말 것) |

5. 서버를 다시 켜고 http://127.0.0.1:8000/health 확인

**[스크린샷]** 복구 후 `/health` 정상
