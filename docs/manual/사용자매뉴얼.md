## 1. 이 프로그램이 뭔가요?

Kiki Trade AI는 **주식(키움)** 과 **암호화폐(업비트)** 정보를 모아,  
후보 종목 찾기 · AI 분석 · 위험 관리 · 주문까지의 흐름을 돕는 **자동매매 플랫폼**입니다.

### 구성 (그림으로 보면)

```text
내 브라우저 (관리 화면)
        ↓
   서버(백엔드)  ←→  데이터베이스(저장소)
        ↓
  키움 / 업비트 / AI(Ollama)
```

| 쉬운 말 | 주소(기본값) | 역할 |
|---------|--------------|------|
| 관리 화면 | http://localhost:3000 | 눈으로 보는 화면 |
| 서버 | http://127.0.0.1:8000 | 실제 계산·연동 |
| API 설명서 | http://127.0.0.1:8000/docs | 서버 기능을 버튼으로 시험 |

**실전 주문(진짜 돈)** 은 기본으로 **꺼져 있습니다.**  
연습(모의)부터 쓰도록 만들어 두었습니다.

설치가 안 되어 있으면 → [설치매뉴얼.md](설치매뉴얼.md)  
서버를 켜는 방법 → [운영매뉴얼.md](운영매뉴얼.md)

## 2. 쓰기 전에 준비할 것

1. 컴퓨터에 설치가 끝난 상태 ([설치매뉴얼.md](설치매뉴얼.md))

## 3. 관리 화면 열기 · 입장하기

### 3-1. 주소 열기

브라우저 주소창에 입력합니다.

```text
http://localhost:3000
```

처음이면 **로그인(입장) 화면**으로 이동합니다.

**[스크린샷]** 로그인(입장) 화면 전체

### 3-2. 입장 방법 (지금 버전)

| 질문 | 답 |
|------|-----|
| 아이디·비밀번호가 있나요? | **없습니다.** (아직 만들지 않음) |
| 어떻게 들어가나요? | **「개발 모드로 입장」** 버튼을 누릅니다 |

버튼을 누르면 대시보드로 이동합니다.  
(이 PC 안에서만 쓰는 간단한 입장 방식입니다.)

**[스크린샷]** 「개발 모드로 입장」 버튼이 보이는 곳

> 전문 용어: **JWT 로그인** = 아이디·비밀번호로 발급하는 출입증 방식.  
> 이 제품의 관리 화면에는 아직 없습니다.

## 4. 대시보드 보기

입장 후 첫 화면이 **Dashboard(대시보드)** 입니다.

### 여기서 볼 수 있는 것

- 환영·안내 문구
- **서버 연결 상태** (연결됨 / 끊김)  
  → 서버의 “건강 점검(`/health`)”을 화면이 대신 확인합니다
- 기초 체크리스트 (연결 여부 등)

### 여기서 아직 안 보이는 것

- 계좌 잔고
- 주문하기 버튼
- AI 추천 카드

**[스크린샷]** 대시보드 전체 (서버 연결됨 상태)

## 8. 문제가 생기면

1. 서버가 켜져 있는지 확인 (`/health`)
2. 관리 화면을 새로고침
3. 그래도 안 되면 [장애대응매뉴얼.md](장애대응매뉴얼.md)
4. 자주 묻는 질문: [FAQ.md](FAQ.md)

관리 화면에 **로그(기록) 전용 화면은 없습니다.**  
서버를 켠 검은 창(터미널)에 메시지가  manifest습니다.
