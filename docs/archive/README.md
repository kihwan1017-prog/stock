# docs/archive

## Why archived

Past step instructions and superseded apply guides are kept here so history is recoverable without cluttering the current domain docs.

**Do not use archive docs for daily install/ops.** Prefer:

- [../deployment/](../deployment/)
- [../trading/](../trading/)
- [../database/](../database/)
- [../reference/](../reference/)

## Layout

| Path | Purpose |
|------|---------|
| [steps/](steps/) | `README_STEP*` work logs only + STEP01–99 catalog |
| [notes/](notes/) | Apply memos (`APPLY_NOTES`) |
| [obsolete/](obsolete/) | Superseded guides (e.g. STEP34 numbered apply pack) |

## Policy

- No deletion of markdown content.
- New STEP work logs go to `steps/` only.
- Completed roadmaps stay in `../reference/`, not here.
